Modify pom plugin:

<build>
            <!-- Plugin to copy the content of /client/dist/ directory to output directory (ie/ /target/pec/) -->
            <!-- This also does filtering on *.js and *.map by replacing @pec.carma.url@ with a string -->
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-resources-plugin</artifactId>
                <version>3.0.2</version>
                <executions>
                    <execution>
                        <id>default-copy-resources</id>
                        <phase>process-resources</phase>
                        <goals>
                            <goal>copy-resources</goal>
                        </goals>
                        <configuration>
                            <overwrite>true</overwrite>
                            <outputDirectory>${project.build.directory}/${project.artifactId}-${project.version}/</outputDirectory>
                            <resources>
                                <resource>
                                    <directory>${project.basedir}/client/dist</directory>       <— this resource copies ALL files to outputDirectory —>
                                </resource>
                                <resource>
                                    <directory>${project.basedir}/client/dist</directory>       <— this resource filters and replaces any variables between @@ in files ending with .js or .map —>
                                    <filtering>true</filtering>
                                    <includes>
                                        <include>**/*.js</include>
                                        <include>**/*.map</include>
                                    </includes>
                                </resource>
                            </resources>
                        </configuration>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>

I also combine this with a default property … the example here is the variable name @pec.carma.url@

    <properties>
        <pec.carma.url>https://lpv-hwdevapp01.co.ihc.com:9217/carma-war/services/protocols/</pec.carma.url>
    </properties>

So any .js or .map file with gets this string from the property pec.carma.url replaced wherever @pec.carma.url@ is specified in the .js/.map files.
You can override this at the command line by doing -Dpec.carma.url=“https://blah.de:9217/burp/blue/reds/”
