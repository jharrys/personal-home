* Setup your datasource
    * View -> Tool & Windows -> Database
    * Configure (wrench icon)
        * Oracle - Add JDBC driver
    * + add a datasource
        * for ldap JDBC url - it works in Intellij ... just use URL
* Configure module for JPA/JEE/EJB framework (facet)
    * Go to project tool window
    * right-click on module & select "Add Framework Support"
    * select JPA JEE
    * choose hibernate implementation
    * after creation you can choose which persistence.xml to use if it already exists for the module
    * if problem using a specific version of jpa, the dependency loading may be conflicting
        * go to module settings
        * click on "dependencies" tab
        * re-order so that jpa 2.1 is higher than the rest of the dependencies

**NOTE**
We do not use a persistence.xml file as we configure this in the spring xml files. Intellij requires you to use a persistence.xml file (I haven’t found a work-around YET); in the meantime you can create minimal persistence.xml file under project/META-INF/persistence.xml. All it needs is:

        <?xml version="1.0" encoding="UTF-8"?>
        <persistence xmlns="http://xmlns.jcp.org/xml/ns/persistence" version="2.1">
            <persistence-unit name="myPersistenceUnit" transaction-type="RESOURCE_LOCAL">
                <properties>
                    <!– Scan for annotated classes and Hibernate mapping XML files –>
                    <property name="hibernate.archive.autodetection" value="class" />
                </properties>
            </persistence-unit>
        </persistence>
