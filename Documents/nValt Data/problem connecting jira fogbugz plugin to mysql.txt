When trying to migrate fogbugz to jira, and using jira's fogbugz plugin to connect to mysql. Got the following error:

        Error connecting to the database: Could not create connection to database server. Attempted reconnect 3 times. Giving up.
        
1. Had copied the mysql connector to JIRA_INSTALL/lib
2. Restarted jira server
3. Tried connecting to mysql fogbugz, but got error (re-checked jdbc connection info)

from jira log file

**Fixed**
In the advanced section of the jira fogbugz plugin make sure the url has:

        autoReconnect=true&useUnicode=true&zeroDateTimeBehavior=convertToNull&useCursorFetch=true&defaultFetchSize=100&sessionVariables=storage_engine=InnoDB
        
*sessionVariables=storage_engine=InnoDB* is the important part
